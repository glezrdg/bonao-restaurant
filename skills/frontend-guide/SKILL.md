## Responsive Image Implementation

### Next.js Image Component Pattern

```tsx
import Image from 'next/image';

// Hero image (above the fold, priority load)
<Image
  src="/optimized/restaurant/_S8A7268_1920.webp"
  alt="Bonao Restaurant dining room with warm ambient lighting"
  width={1920}
  height={1080}
  priority
  className="object-cover"
  sizes="100vw"
/>

// Food card image (below fold)
<Image
  src="/optimized/food/BONAO BURGER .2_800.webp"
  alt="Bonao Burger with purple cabbage slaw and fries"
  width={800}
  height={600}
  className="object-cover"
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
/>
```

### Manual srcset Pattern (HTML)

```html
<picture>
  <source
    type="image/webp"
    srcset="
      /optimized/food/GUAVA CHIPOTLE RIBS_400.webp 400w,
      /optimized/food/GUAVA CHIPOTLE RIBS_800.webp 800w,
      /optimized/food/GUAVA CHIPOTLE RIBS_1200.webp 1200w,
      /optimized/food/GUAVA CHIPOTLE RIBS_1920.webp 1920w
    "
    sizes="(max-width: 400px) 400px,
           (max-width: 800px) 800px,
           (max-width: 1200px) 1200px,
           1920px"
  />
  <img
    src="/optimized/food/GUAVA CHIPOTLE RIBS_800.webp"
    alt="Guava Chipotle Ribs with crispy onions"
    loading="lazy"
  />
</picture>
```

### Size Selection Guidelines

| Use Case | Recommended Size | Reasoning |
|----------|------------------|-----------|
| Full-width hero | `_1920` | Maximum quality for hero |
| Hero with overlay | `_1200` | Overlay reduces detail need |
| Two-column split | `_1200` | Half viewport width |
| Three-column grid | `_800` | ~33% viewport width |
| Four-column grid | `_800` | Still sharp at 25% viewport |
| Thumbnail/card | `_400` | Small previews |
| Mobile full-width | `_800` | Good balance |
| Gallery lightbox | `_1920` | Full detail on click |

## Image Optimization Best Practices

### Performance Rules

1. **Always use `priority` for above-the-fold images** (hero, first visible content)
2. **Use `loading="lazy"` for below-fold images** (Next.js does this by default)
3. **Specify explicit `width` and `height`** to prevent layout shift (CLS)
4. **Use appropriate `sizes` attribute** for responsive loading

### Alt Text Guidelines

Write descriptive, contextual alt text:

```tsx
// Good
alt="Bonao Burger topped with braised meat, purple cabbage slaw, served with crispy fries"
alt="Bonao Restaurant dining room featuring warm lighting and signature wooden face sculpture"

// Bad
alt="burger"
alt="restaurant"
alt="food image"
```

### File Path Handling

Some filenames contain spaces and special characters. Handle with encoding:

```tsx
// Option 1: URL encode spaces
src="/optimized/food/BONAO%20BURGER%20.2_800.webp"

// Option 2: Use encodeURIComponent in dynamic paths
const imagePath = encodeURIComponent("BONAO BURGER .2");
src={`/optimized/food/${imagePath}_800.webp`}
```

## Quick Reference: File Naming Pattern

```
[DISH NAME]_[SIZE].webp

Where SIZE is one of:
- 400  (thumbnail, mobile small)
- 800  (cards, mobile full)
- 1200 (desktop columns)
- 1920 (hero, full-bleed)
```

Examples:
- `BONAO BURGER .2_800.webp`
- `_S8A7268_1920.webp`
- `GUAVA CHIPOTLE RIBS_1200.webp`
